Using an Arduino to Program an AVR
by zagGrad | September 16, 2010 | 44 comments Skill Level: Beginner

Reprogramming AVRs
 

Many of the products SparkFun sells use an AVR, like an ATmega168 or an ATmega328, as a microprocessor. There are many reasons we like this microprocessor, not the least of which is that it is the processor used in Arduino. Just because a board uses an ATmega, though, doesn't mean it's an Arduino. However it does mean that code written for an Arduino can be used on a board that uses the same microprocessor! The only thing standing in the way of running custom code on a board with an AVR is actually loading the code onto the chip (since it likely isn't loaded with the serial bootloader used on an Arduino). This tutorial is a guide on how to use an Arduino as a programmer to load code onto a standalone ATmega microcontroller.

Required Software
 

  • Arduino 0018 (or newer)
  • WinAVR (Windows) or Crosspack for AVR Development (Mac) - This software contains avrdude, which is needed! (Note: You could get around downloading this and using the copy of avrdude that comes with Arduino; however installing the version with WinAVR will add the path to the Windows Envirnonment Variables which makes it easier to use in the command prompt)

Hardware Required
 

  • Arduino Duemilanove (or similar)
  • Target Board (for this tutorial the RGB Matrix with Serial Interface will be used)
  • 6 F/F Jumper Wires
  • 12 Breakaway Male Headers in 3x segments
  • Soldering Iron/Solder

Firmware Required
 

  • ArduinoISP Sketch (Included in the 'Examples' folder of Arduino)
  • Hex File for Target Board (Don't know how to get this? Don't worry, it's covered in the tutorial)

Getting the Hardware Ready
 

Arduino uses a serial bootloader to load code onto the AVR; this is great for ease of use as it only requires a USB cable to program, however it has some drawbacks too. The serial bootloader 'permanently' takes up code space, and it also limits the programming speed which is why very large sketches can take a while to load. On most of the boards that are created at SparkFun the code is loaded using the ISP header. ISP stands for In-System Programmer; basically it allows the microprocessor to be programmed via its SPI lines. If you've ever had your hands on a board that uses an AVR, you've likely seen a header that looks like this:


ISP Header in Lower Right Corner

This is the ISP header, and it provides access to all the pins needed to program the AVR. You can even see the ISP header on the Arduino Duemilanove boards. Several pins from an Arduino can be connected to another board's ISP signals, allowing the board to be programmed by the Arduino.


Male Headers Soldered Into ISP Header of Arduino and Target Board

After the headers are installed the Arduino needs to be wired to the ISP connector on the target board. The ISP connector consists of 6 signals: Vcc, GND, Reset, SCK, MOSI and MISO. The picture below illustrates which pins belong to which position on the header.


Labeled ISP Header

Pin 1 of the ISP header may have different notations from different manufacturers. SparkFun uses a line drawn over pin 1, while Arduino actually prints a '1' by pin 1. The following table illustrates how the Arduino should be wired up to the target board. It also lists the pin number for the Arduino board in case the pin headers weren't soldered onto the Arduino ISP header.
 

Arduino Pin Number Arduino ISP Pin Target ISP Pin
5V Vcc Vcc
Gnd Gnd Gnd
D11 MOSI MOSI
D12 MISO MISO
D13 SCK SCK
D10 D10 Reset


Generating the hex file
 

When an Arduino sketch is compiled and uploaded to a board, the Arduino IDE actually creates a new file called a hex file which is sent through the USB cable to the Arduino. The compiler takes the sketch and converts it into a sequence of 1's and 0's that the microcontroller can understand; this sequence of 1's and 0's is the hex file. In order to program the target board we'll need to have access to the hex file for our sketch (you can also use a hex file that has been compiled from WinAVR and GCC if you'd like). For this tutorial a hex file for the RGB Serial Matrix will be used.

It's a bit tricky to access the hex file for an Arduino sketch. The easiest way is to hold the 'shift' key and press the 'Verify' button. If your code compiles correctly, the second to last line will show you where the hex file is located. You'll know the correct location is being shown if the end of the line reads "sketchname.cpp.hex." For example, if I open Blink sketch and press shift while clicking Verify I get this output:

C:\DOCUME~1\rowens\LOCALS~1\Temp\build2695350595880978373.tmp\Blink.cpp.hex

Copy this location as we'll need it in a bit.

Programming the Arduino to act as a Programmer
 

There's one more important step before the target board can be programmed with the Arduino: the Arduino has to be programmed to work as a programmer! This is very simple; with the Arduino IDE open, open the ArduinoISP sketch by going to File->Examples->ArduinoISP. This sketch will make the Arduino act as an ISP programmer. Select your board in the Tools menu, and make sure the correct COM port is selected (also, copy down or make a mental note of the COM port. It will be needed in the next step). Then just upload the sketch to the Arduino.

Programming the Target Board
 

By now, the Arduino is programmed to act as an ISP programmer, the Arduino is connected to the target board and plugged into a USB cable from the computer, and a hex file is ready to be uploaded to the target board. The only thing left to do is send the hex file from the computer to the Arduino so that it can send the program to the target board. Avrdude is a software program that comes with WinAVR (or Crosspack) that can send a hex file to an ISP programmer. Once you've got avrdude installed (via WinAVR or Crosspack) you're only a command away from getting the hex file to the target board. Installing these programs is simple, just follow all the prompts and install the programs with their default settings.

Avrdude is a command line program, so you'll have open a command prompt in order to use it.  Once the command line is opened, type avrdude and press enter; if it's installed correctly then a list of available options will be listed on the screen. Now we're ready to send a command that will program the board. Start by navigating to the directory that the hex file was placed in. The directory can be found be removing the file name from the path that you copied in the "Generating a Hex File" section. For example, in the command prompt I had to enter the command:

cd C:\DOCUME~1\rowens\LOCALS~1\Temp\build2695350595880978373.tmp\

Now enter this command, replacing comport with the comport that your  Arduino is plugged into, processor for the processor you're programming, and hexfilename for the name of your file:

avrdude -P comport -b 19200 -c avrisp -p processor -v -e -U flash:w:hexfilename.hex

Don't forget the command is case-sensitive, so be careful typing it or an error may occur. The command uses seven parameters: -P is the comport used, -b is the baudrate, -c is the programmer type, -p is the microprocesor being programmed, -v tells avrdude to use a verbose output, -e tells avrdude to erase the chip before programming it, and -U tells avrdude where to put the hex file.

The processor parameter can be a bit confusing sometimes. If you're programming an ATmega328, then m328p should be used for the processor parameter. If you're programming an ATmega168, m168 should be used. You can find the appropriate parameter for any AVR by typing 'avrdude -p?' at the command prompt to get a giant list of possibilities.


Avrdude Output

Recap
 

That's it! Now you know how to program a standalone AVR from an Arduino using ArduinoISP. Here's a shortened version to recap what was covered:

  1. Upload ArduinoISP to the Arduino
  2. Connect the Arduino to the Target Board's ISP signals
  3. Generate the hex file to be loaded onto the Target Board
  4. Use avrdude to send the hex file to the Arduino, which will program the Target Board

Hope you learned something! Leave questions or improvements in the comments section.
